Quickstart

Build OS in 3 minutes

  1. Open the Build OS tab.
  2. Create a project with title, problem statement, and target user.
  3. Fill each stage artifact and advance only when gates pass.
Stage progression is strict: only the current stage can be advanced.
Workflow

5-stage canonical Eli loop

explore

Explore

Experiment with APIs/tools and validate there is enough signal to spec.

prd_research

PRD + Research

Define product requirements and market signals before engineering handoff.

tdd_review

TDD Review

Pressure-test technical design until open engineering questions are closed.

build_loop

Build Loop

Implement, deploy, and run closed-loop QA until the spec is met.

manual_polish

Manual Polish

Final pass for polish; finish with status shipped or parked.

Artifacts + Gates

Required docs and gate logic

Explore

Required: experiment_log

Gate: experiment log not empty and project problem statement is present.

PRD + Research

Required: prd, market_signals

Gate: both docs non-empty and research checklist item marked complete.

TDD Review

Required: tdd, engineering_questions

Gate: TDD non-empty and open engineering questions resolved checkbox checked.

Build Loop

Required: implementation_notes, qa_notes

Gate: both docs non-empty, deploy URL present, and spec_passes_qa checked.

Manual Polish

Required: manual_test_notes, polish_backlog

Gate: manual test notes non-empty and status is shipped or parked.

How-To

Operational recipes

How to start a new idea

  1. Create project.
  2. Fill Explore artifact.
  3. Advance stage.

How to close a build cycle

  1. Complete Build Loop notes and deploy URL.
  2. Run QA and check spec_passes_qa.
  3. Advance to Manual Polish.

How to finish a project

  1. Complete manual notes.
  2. Set status to shipped or parked.
  3. Click Finish Project.

How to copy the exact AI packet

  1. Select current stage.
  2. Review Prompt Packet panel.
  3. Click Copy.

Troubleshooting

Common failures and fixes

  • stage_gate_failed: required artifact/checklist/status is missing.
  • Only the current stage can be updated: switch to current stage rail node.
  • Refresh token is not valid: rotate SMOKE_SUPABASE_REFRESH_TOKEN.
  • Function auth errors: deploy Build OS functions with --no-verify-jwt.
